*[CVE]: Common Vulnerabilities and Exposures
*[CWE]: Common Weakness Enumeration
*[CVSS]: Common Vulnerability Scoring System
*[TTP]: Tactics, Techniques, and Procedures — adversary behavioral patterns per MITRE ATT&CK
*[MITRE ATT&CK]: A knowledge base of adversary tactics and techniques based on real-world observations
*[PoC]: Proof of Concept — a working exploit demonstrating a vulnerability is real and exploitable
*[SAST]: Static Application Security Testing — analysis of source code without execution
*[DAST]: Dynamic Application Security Testing — analysis of running applications
*[SSDF]: Secure Software Development Framework — NIST framework for secure development practices
*[SLSA]: Supply-chain Levels for Software Artifacts — a security framework for supply chain integrity
*[OpenSSF]: Open Source Security Foundation
*[P-SSCRM]: Product-Security Software Supply Chain Risk Management task framework
*[RemOps]: Remediation Operations — the practice of orchestrating and automating vulnerability remediation at scale
*[LLM]: Large Language Model
*[MTTR]: Mean Time to Remediate — average time between vulnerability discovery and resolution
*[ASPM]: Application Security Posture Management
*[EDR]: Endpoint Detection and Response
*[SBOM]: Software Bill of Materials
*[CI/CD]: Continuous Integration / Continuous Deployment
*[Source-to-Sink]: Data flow analysis tracing untrusted input (source) through code to a dangerous operation (sink)
