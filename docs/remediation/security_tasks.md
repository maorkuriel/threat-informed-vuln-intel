# Security Tasks

!!! abstract "Overview"
    Security Tasks are TIVI's developer-facing remediation unit — grouping related vulnerabilities into a single actionable work item with context, a concrete fix, and compliance requirements pre-attached.

## Task Schema

```python
@dataclass
class SecurityTask:
    task_id: str
    title: str                         # Human-readable outcome, not CVE numbers
    security_outcome: str              # What fixing this achieves
    findings: list[Finding]           # All CVEs/findings grouped into this task
    ttp_techniques: list[str]          # MITRE ATT&CK techniques addressed
    compliance: list[ComplianceReq]   # NIST SSDF / SLSA / OpenSSF requirements
    fix: GeneratedFix                 # AI-generated code change
    pr_url: str | None                # Link to prepared PR (if auto-created)
    priority: int                     # 0-5 (inherits from highest-priority finding)
    estimated_effort_hours: float
    business_justification: str       # Why this matters in business terms
```

## Grouping Logic

TIVI groups findings into tasks based on:

1. **Same ATT&CK technique** — findings that enable the same attack technique belong in one task
2. **Same package/component** — multiple CVEs in the same dependency = one upgrade task
3. **Same remediation action** — findings fixed by the same code change are grouped
4. **Same compliance requirement** — findings satisfying the same NIST control are grouped

```python
def group_into_tasks(findings: list[Finding]) -> list[SecurityTask]:
    groups = defaultdict(list)
    
    for finding in findings:
        # Primary grouping: by remediation action
        key = (finding.affected_package, finding.remediation_action_type)
        groups[key].append(finding)
    
    tasks = []
    for (package, action_type), group_findings in groups.items():
        task = SecurityTask(
            title=generate_task_title(group_findings),
            findings=group_findings,
            ttp_techniques=union_ttp(group_findings),
            compliance=union_compliance(group_findings),
            fix=generate_fix(group_findings),
            priority=max(f.priority for f in group_findings),
            estimated_effort_hours=estimate_effort(group_findings)
        )
        tasks.append(task)
    
    return sorted(tasks, key=lambda t: t.priority)
```

## Business Justification

Every task includes a business-context summary generated by TIVI:

```
This task addresses 4 vulnerabilities that collectively enable
T1195 (Supply Chain Compromise) — the technique used in the
SolarWinds and XZ Utils attacks.

Fixing this prevents unauthorized modification of your build
artifacts, protects your customers from downstream compromise,
and satisfies SLSA Level 2 requirements for your software supply
chain compliance posture.

Estimated developer effort: 2 hours.
Estimated risk reduction: High (4 confirmed exploitable findings closed).
```

!!! tip
    The business justification is specifically designed to address the 68% developer disengagement problem. Developers who understand *why* a security task matters act on it. Developers who receive a CVE number don't.
